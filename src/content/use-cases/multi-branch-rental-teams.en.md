This use case fits rental operators that work across several branches and cannot allow quoting to drift away from actual branch conditions.

## What makes multi-branch demand harder

The challenge is not simply having more locations. The challenge is that each location changes what the next valid action should be.

- one branch may fit the request while another does not
- vehicle availability can look similar until local context is checked
- managers waste time preparing options that were never branch-realistic
- approvals become slower because the case arrives without enough operating clarity

Without branch-aware intake, the team ends up correcting the workflow after the fact.

## What this use case needs from the system

For a multi-branch team, the system should keep branch relevance visible from the first interaction:

1. connect the lead to branch intent early
2. hold availability inside branch context instead of generic inventory language
3. prepare pricing and reservation steps only after the branch picture is credible
4. hand off to the manager with enough context to act instead of reconstructing the case

## Why this deserves its own page

For multi-branch operators, branch awareness is not a feature add-on. It is the condition that keeps the quote path operationally honest.
